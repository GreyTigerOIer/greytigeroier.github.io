Pollard-Rho算法浅谈

发布于 2020-03-25 | 分类于 数论 、 OI 、 因数分解 | 6分钟 | 1365字数

这是我写的第一篇博客

最初版本发布于博客园，此版本为修改版

介绍

Pollard-Rho算法是Pollard在1975年提出的一种大整数因数分解算法
由于利用了伪随机数的内在循环，因此算法名有后缀Rho

Pollard-Rho的时间复杂度O(n4)O(\sqrt[4]{n})O(4n​)在大整数因数分解算法中不算出众，但其空间复杂度O(1)O(1)O(1)与低代码难度使之成为OI界的一种常见算法

原理

  • Q：随机找一个数xxx，满足x∣n&x>1x\mid n\And x > 1x∣n&x>1的概率有多大？
    A：d(n)n=Ω(n−1)\frac{d(n)}{n}=\Omega(n^{-1})nd(n)​=Ω(n−1)
  • Q：随机找一个数yyy，它和gcd⁡(y,n)∣n&gcd⁡(y,n)>1\gcd(y,n)\mid n\And \gcd(y,n) > 1gcd(y,n)∣n&gcd(y,n)>1有共同因子的概率有多大？
    A：n−Φ(n)n=Ω(n−12)\frac{n-\Phi(n)}{n}=\Omega(n^{-\frac{1}{2}})nn−Φ(n)​=Ω(n−21​)

比较上面两个式子，发现猜yyy猜出因子的概率比猜xxx更大，于是我们可以随机选择一个yyy，那么gcd⁡(y,n)\gcd(y,n)gcd(y,n)就是我们找出的一个因子。

我们又可以想这样一个悖论：在mmm个数中重复地选择m\sqrt{m}m​个数，那么就有>12>\frac{1}{2}>21​的概率有两个数相等，这就是“生日悖论”。

回到算法，我们可以随机选择n4\sqrt[4]{n}4n​个数，然后两两相减，再把结果与nnn求gcd⁡\gcdgcd，但这样时间复杂度又回到了O(n)O(\sqrt{n})O(n​)，有什么方法能既保证随机，又有一定规律，能快速走到重合位置呢？使用伪随机数！

伪随机数能保证随机，又由于“生日悖论”，伪随机数的循环大小一般不会超过n4\sqrt[4]{n}4n​（因为如果出现两个数相同，那么就说明我们走到了原来走过的位置），而只要找到这个循环（相当于我们走到了我们原来走过的位置，使得p∣y0−y1p\mid y_0-y_1p∣y0​−y1​），我们就找到了一个因子。

为了保证伪随机数的状态数与nnn同阶，从而使“生日悖论”生效，因此一般采用f(x)=(x2+c) mod nf(x)=(x^2+c)\bmod nf(x)=(x2+c)modn

如何找到这个循环呢？这就引入了Pollard-Rho算法的不同实现了。

  • Floyd找环法：这个方法可以用跑步时的“套圈”来比喻.
    维护两个伪随机数x,yx,yx,y，一开始x0=y0x_0=y_0x0​=y0​（起点相同）。
    然后让这两个随机数以不同速度转移xw+1=f(xw),yw+1=f(f(yw))x_{w+1}=f(x_w),y_{w+1}=f(f(y_w))xw+1​=f(xw​),yw+1​=f(f(yw​))，直到第二次xt=ytx_t=y_txt​=yt​。
    由于这两个随机数有“速度差”，因此一定会有一个时刻xxx会被yyy“套圈”。
  • Brent找环法：Floyd找环法最大的问题在于转移次数过多，如果yyy比xxx更慢，又能及时“追上”xxx，不被xxx落在循环外面就好了。
    于是我们考虑倍增环长的办法，令yw=xmax⁡{2t:2t<w}y_w=x_{\max\{2^t:2^t<w\}}yw​=xmax{2t:2t<w}​，由于xxx是随机数，那么yyy实际上也是随机的。
    而当走到环要ppp步，环的长度为qqq时，则需要min⁡w{w:w−q=2t≥p}\min_{w}\{w:w-q=2^t\ge p\}minw​{w:w−q=2t≥p}步，可以发现，所需要的步数也是O(p+q)O(p+q)O(p+q)的，这就证明了我们能够通过这种方法在线性时间内找到环。

这样，我们就能列出以下的伪代码了：

int x = p_0, y = p_0, i = 0;
do {
    if(Floyd) y = f(f(y));
    else if((i & (-i)) == i) y = x;
    x = f(x), i++;
    int g = gcd(abs(x, y), n);
    if(g > 1 and g < n) return g;
} while(x != y);

优化

我们发现现在的代码效率远远不够，仔细分析发现瓶颈在于过多的gcd⁡\gcdgcd操作使得时间复杂度变为O(n4log⁡n)O(\sqrt[4]{n}\log n)O(4n​logn)，考虑如何删去这个log⁡n\log nlogn。

由于(x−y)⊥n(x-y)\perp n(x−y)⊥n的概率很高，因此我们可以不一个个去试，而是将若干个(x−y)(x-y)(x−y)乘起来，再一起取一个gcd⁡\gcdgcd，这样的话，每次gcd⁡\gcdgcd找出因子的概率就大大增加了

实际上这种优化是用gcd⁡=n\gcd=ngcd=n的概率增加换gcd⁡=1\gcd=1gcd=1的概率减小，因此选乘起来的(x−y)(x-y)(x−y)对的总数就显得尤为重要，在实际使用中我们会一般使用100∼300100\sim 300100∼300个来乘。

伪代码

//p_0：循环开始点；m：(x-y)累乘次数
//isprime(n)：Miller-Rabin算法实现的素性判断
int pollard_rho(int n) {
    if(isprime(n)) return n;
    #define f(x) (((x) * (x) + c) % n)
    for(int c = rand() % n + 1; ; c++) {
        int x = p_0, y = p_0, tot = 1, i = 0;
        do {
            if(Floyd) y = f(f(y));
            else if((i & (-i)) == i) y = x;
            x = f(x), i++;
            tot  = tot * abs(x, y) % n;
            if(i % m == 0) {
                int g = gcd(tot, n);
                if(g > 1 and g < n) return g;
                tot = 1;
            }
        } while(x != y);
        int g = gcd(tot, n);
        if(g > 1 and g < n) return g;
    }
}
